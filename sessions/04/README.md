# جلسه 4 - مدیریت کاربران و سطوح دسترسی

## تمرین‌های عملی در کلاس و پاسخ‌های کامل

### تمرین 1: ایجاد فایل و تنظیم مجوزهای خاص
یک فایل به نام `report.docx` بسازید. مجوزهای آن را طوری تنظیم کنید که:
- مالک بتواند آن را بخواند و بنویسد
- اعضای گروه فقط بتوانند آن را بخوانند
- دیگران هیچ دسترسی نداشته باشند

#### پاسخ (حالت عددی)
```bash
touch report.docx
chmod 640 report.docx
```

#### پاسخ (حالت نمادین)
```bash
touch report.docx
chmod u+rw report.docx      # مالک: خواندن و نوشتن
chmod g+r report.docx       # گروه: فقط خواندن
chmod o-rwx report.docx     # دیگران: بدون دسترسی
```

یا به‌طور یکپارچه:
```bash
chmod u=rw,g=r,o= report.docx
```

#### توضیح
- `640` به معنی:
  - `6` (مالک): 4 + 2 = rw- (خواندن و نوشتن)
  - `4` (گروه): فقط r- (خواندن)
  - `0` (دیگران): --- (بدون دسترسی)

#### تأیید نتیجه
```bash
ls -l report.docx
# خروجی: -rw-r----- ... report.docx
```

### تمرین 2: تنظیم مجوزهای دایرکتوری اشتراکی
یک دایرکتوری به نام `shared` بسازید. مجوزهای آن را طوری تنظیم کنید که:
- مالک و اعضای گروه بتوانند وارد آن شده، فایل‌ها را ببینند و فایل جدید ایجاد کنند
- دیگران فقط بتوانند وارد شده و لیست فایل‌ها را ببینند

#### پاسخ (حالت عددی)
```bash
mkdir shared
chmod 750 shared
```

#### پاسخ (حالت نمادین)
```bash
mkdir shared
chmod u=rwx shared        # مالک: دسترسی کامل
chmod g=rx shared         # گروه: خواندن و ورود (بدون نوشتن)
chmod o=rx shared         # دیگران: خواندن و ورود
```

یا به‌طور یکپارچه:
```bash
chmod u=rwx,g=rx,o=rx shared
```

#### توضیح
- `750` به معنی:
  - `7` (مالک): 4 + 2 + 1 = rwx (دسترسی کامل)
  - `5` (گروه): 4 + 0 + 1 = r-x (خواندن و ورود)
  - `0` (دیگران): --- (بدون دسترسی)

در واقع پاسخ دستوری سوال مجدداً بررسی می‌شود: دیگران فقط می‌توانند وارد شوند و لیست ببینند، بنابراین:
```bash
chmod 750 shared
```

یا اگر دیگران هم دسترسی داشته باشند:
```bash
chmod 755 shared
```

#### تأیید نتیجه
```bash
ls -l | grep shared
# خروجی: drwxr-x--- ... shared
```

## تمرین‌های هفته آینده و پاسخ‌های تفصیلی

### سوال 1: تبدیل مجوز نمادین به عددی
مجوز `---rwxr-x` معادل چه عددی در سیستم Octal است؟

#### پاسخ
`071`

#### توضیح مفصل
```
--- rwx r-x
0   7   5

- Owner (---): 0 (بدون دسترسی)
  - r: بدون = 0
  - w: بدون = 0
  - x: بدون = 0
  - مجموع: 0

- Group (rwx): 7 (دسترسی کامل)
  - r: بله = 4
  - w: بله = 2
  - x: بله = 1
  - مجموع: 4 + 2 + 1 = 7

- Others (r-x): 5 (خواندن و اجرا)
  - r: بله = 4
  - w: بدون = 0
  - x: بله = 1
  - مجموع: 4 + 0 + 1 = 5

نتیجه: 075
```

**نکته**: برخی سیستم‌ها می‌توانند یک رقم سوم در ابتدا اضافه کنند (برای special permissions)، پس `075` یا `0075` است.

### سوال 2: حذف مجوز اجرا از گروه
دستوری بنویسید که مجوز اجرای یک اسکریپت به نام `deploy.sh` را فقط از گروه آن حذف کند، بدون اینکه مجوزهای دیگر تغییر کند.

#### پاسخ
```bash
chmod g-x deploy.sh
```

#### توضیح
در جلسه یاد گرفتیم که حالت نمادین `chmod` اجازه می‌دهد تنها مجوزهای خاصی را تغییر دهیم:
- `g`: گروه
- `-`: حذف مجوز
- `x`: اجرا

این دستور فقط مجوز اجرا را از گروه حذف می‌کند و بقیه مجوزها تغییر نمی‌کند.

#### مثال
```bash
# قبل: -rwxr-xr-x (755)
chmod g-x deploy.sh
# بعد:  -rwxr--r-x (754)
```

### سوال 3: محاسبه مجوزهای پیش‌فرض با umask
اگر `umask` سیستم `0027` باشد، مجوز پیش‌فرض برای یک فایل جدید و یک دایرکتوری جدید چه خواهد بود؟

#### پاسخ

**برای فایل جدید:**
```
666 - 027 = 640
نمادین: rw-r-----
```

**برای دایرکتوری جدید:**
```
777 - 027 = 750
نمادین: rwxr-x---
```

#### توضیح مفصل
در جلسه یاد گرفتیم که `umask` از مجوزهای کامل کم می‌شود:

**محاسبه برای فایل:**
```
    666 (مجوز کامل)
  - 027 (umask)
  -----
    640 (نتیجه)

تحلیل:
- Owner (6): 6 - 0 = 6 (rw-)
- Group (6): 6 - 2 = 4 (r--)
- Others (6): 6 - 7 = -1 → 0 (---)

نتیجه: 640 (rw-r-----)
```

**محاسبه برای دایرکتوری:**
```
    777 (مجوز کامل)
  - 027 (umask)
  -----
    750 (نتیجه)

تحلیل:
- Owner (7): 7 - 0 = 7 (rwx)
- Group (7): 7 - 2 = 5 (r-x)
- Others (7): 7 - 7 = 0 (---)

نتیجه: 750 (rwxr-x---)
```

#### تأیید عملی
```bash
# نمایش umask فعلی
umask
# تغییر موقت umask
umask 0027
# ایجاد فایل و دایرکتوری
touch testfile
mkdir testdir
# بررسی مجوزهای ایجاد شده
ls -l testfile testdir
```

## خلاصه دستورات مهم

| دستور | مثال | توضیح |
|-------|------|-------|
| `chmod` (عددی) | `chmod 755 file` | تنظیم مجوزهای دقیق |
| `chmod` (نمادین) | `chmod u+x file` | اضافه کردن یا حذف مجوز |
| `chown` | `sudo chown user:group file` | تغییر مالک و گروه |
| `chgrp` | `sudo chgrp group file` | تغییر گروه |
| `umask` | `umask 0027` | تنظیم مجوزهای پیش‌فرض |

---

## نکات کلیدی

1. **chmod عددی سریع‌تر است** و بیشتر استفاده می‌شود
2. **chmod نمادین دقیق‌تر است** و تنها بخش‌های مورد نیاز را تغییر می‌دهد
3. **umask** مجوزهایی که نداشته باشند را مشخص می‌کند، نه آن‌هایی که دارند
4. **معادله umask**: مجوز نهایی = مجوز کامل - umask
