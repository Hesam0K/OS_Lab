# جلسه 4 - مدیریت کاربران و سطوح دسترسی

## بخش اول: مقدمه و مباحث تئوری

### 1. مرور سریع جلسه قبل
در جلسه قبل با این مفاهیم آشنا شدیم:
- جستجو و کار با محتوای فایل‌ها
- دستورات `grep`، `find`، `head`، `tail`، `wc`
- Wildcards و Regular Expressions
- Piping و Redirection
- جریان‌های استاندارد stdin، stdout، stderr

### 2. چرا مدیریت دسترسی مهم است؟
لینوکس از ابتدا به عنوان یک سیستم‌عامل چندکاربره (Multi-user) طراحی شده است. این به معنی آن است که چندین کاربر می‌توانند به صورت همزمان از یک سیستم استفاده کنند.

برای جلوگیری از تداخل، حفظ حریم خصوصی و امنیت سیستم، باید مشخص شود که:
- هر کاربر به چه فایل‌ها و دایرکتوری‌هایی دسترسی دارد
- چه کارهایی می‌تواند با آن‌ها انجام دهد

اینجاست که سیستم مجوزها (Permissions) اهمیت پیدا می‌کند.

### 3. کاربران و گروه‌ها

#### کاربر (User)
هر کسی که با سیستم تعامل دارد، یک حساب کاربری دارد که شامل:
- نام کاربری (username)
- شناسه منحصر به فرد عددی (UID - User ID)

#### کاربر ریشه (Root)
- نام کاربری: `root`
- UID: 0
- دسترسی کامل و نامحدود به تمام بخش‌های سیستم
- به او "superuser" هم می‌گویند

#### گروه (Group)
- مجموعه‌ای از کاربران
- هر گروه نام و شناسه منحصر به فرد عددی دارد (GID - Group ID)
- سادگی در مدیریت دسترسی
- هر کاربر عضو حداقل یک گروه اصلی (primary group) است و می‌تواند عضو چندین گروه فرعی (secondary groups) نیز باشد

### 4. سیستم مجوزهای فایل در لینوکس

#### تحلیل خروجی `ls -l`

```
-rwxr-x--x  1  ali  developers  1024  Sep 26 09:30  main.c
```

| بخش | توضیح |
|-----|-------|
| `-` | نوع فایل (- = فایل عادی، d = دایرکتوری، l = لینک) |
| `rwxr-x--x` | مجوزها (Permissions) |
| `1` | تعداد hard links |
| `ali` | مالک فایل (Owner/User) |
| `developers` | گروهی که فایل به آن تعلق دارد |
| `1024` | حجم فایل بر حسب بایت |
| `Sep 26 09:30` | تاریخ آخرین تغییر |
| `main.c` | نام فایل |

#### سه سطح دسترسی

**خواندن (Read - r):**
- برای فایل: مشاهده محتوای فایل
- برای دایرکتوری: دیدن فهرست فایل‌های داخل دایرکتوری

**نوشتن (Write - w):**
- برای فایل: تغییر محتوای فایل
- برای دایرکتوری: ایجاد، حذف و تغییر نام فایل‌های داخل دایرکتوری

**اجرا (Execute - x):**
- برای فایل: اجرای فایل (اگر برنامه یا اسکریپت باشد)
- برای دایرکتوری: ورود به دایرکتوری با `cd`

#### تحلیل `rwxr-x--x`

```
rwx | r-x | --x
owner | group | others
```

| بخش | معنی |
|-----|------|
| `rwx` | مالک می‌تواند بخواند، بنویسد و اجرا کند |
| `r-x` | گروه می‌تواند بخواند و اجرا کند، اما نمی‌تواند بنویسد |
| `--x` | دیگران فقط می‌توانند اجرا کنند |

#### نمایش عددی (Octal Notation)

روش سریع‌تر برای نمایش مجوزها:
- `r` (خواندن) = 4
- `w` (نوشتن) = 2
- `x` (اجرا) = 1

جمع این اعداد برای هر دسته:
- `rwx` = 4 + 2 + 1 = 7
- `r-x` = 4 + 0 + 1 = 5
- `--x` = 0 + 0 + 1 = 1
- `rw-` = 4 + 2 + 0 = 6
- `---` = 0 + 0 + 0 = 0

بنابراین `rwxr-x--x` = `751`

## بخش دوم: کار عملی با دستورات

### 1. تغییر مجوزها - chmod

#### حالت نمادین (Symbolic Mode)

```bash
chmod [u|g|o|a] [+|-|=] [r|w|x] filename
```

نمادها:
- `u`: مالک (user)
- `g`: گروه (group)
- `o`: دیگران (others)
- `a`: همه (all)

عملگرها:
- `+`: اضافه کردن مجوز
- `-`: حذف کردن مجوز
- `=`: تنظیم دقیق مجوز

مثال‌ها:
```bash
chmod u+x testfile.txt        # به مالک مجوز اجرا بده
chmod g-w testfile.txt         # از گروه مجوز نوشتن بگیر
chmod o=r testfile.txt         # مجوز دیگران را فقط خواندن تنظیم کن
chmod a+r testfile.txt         # به همه مجوز خواندن بده
```

#### حالت عددی (Octal Mode)

سریع‌تر و رایج‌تر:
```bash
chmod 755 myscript.sh          # rwxr-xr-x
chmod 644 config.conf          # rw-r--r--
chmod 700 private_key          # rwx------
```

### 2. تغییر مالکیت

#### `chown` - تغییر مالک

```bash
sudo chown newuser testfile.txt           # تغییر مالک
sudo chown newuser:newgroup testfile.txt  # تغییر مالک و گروه
```

#### `chgrp` - تغییر گروه

```bash
sudo chgrp newgroup testfile.txt
```

### 3. مجوزهای پیش‌فرض - umask

دستور `umask` مشخص می‌کند که وقتی یک فایل یا دایرکتوری جدید ساخته می‌شود، چه مجوزهایی نداشته باشد.

ماسک از مجوزهای کامل کم می‌شود:
- برای فایل: 666 (rw-rw-rw-)
- برای دایرکتوری: 777 (rwxrwxrwx)

مثال: اگر `umask` برابر `0022` باشد:
- برای فایل: 666 - 022 = 644 (rw-r--r--)
- برای دایرکتوری: 777 - 022 = 755 (rwxr-xr-x)
