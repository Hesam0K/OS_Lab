# جلسه 3 - جستجو و کار با محتوای فایل‌ها

## تمرین‌های عملی در کلاس و پاسخ‌های کامل

### تمرین 1: نمایش 5 خط اول با head
با استفاده از `head`، 5 خط اول فایل `system.log` را مشاهده کنید.

#### پاسخ
```bash
head -n 5 system.log
```

#### توضیح
در جلسه آموختیم که دستور `head` برای نمایش چند خط اول یک فایل به کار می‌رود. آپشن `-n` تعداد خطوط را مشخص می‌کند. بنابراین `head -n 5` به ما 5 خط اول فایل را نشان می‌دهد.

---

### تمرین 2: نمایش 24 خط آخر با tail
با استفاده از `tail`، 24 خط آخر فایل را مشاهده کنید.

#### پاسخ
```bash
tail -n 24 system.log
```

#### توضیح
دستور `tail` برای نمایش چند خط آخر یک فایل استفاده می‌شود. این دستور بسیار برای دیدن آخرین وقایع در فایل‌های لاگ مفید است. آپشن `-n 24` به معنی نمایش 24 خط آخر است.

---

### تمرین 3: جستجو با grep بدون حساسیت به حروف
با `grep` تمام خطوطی که شامل کلمه `network` هستند را پیدا کنید، بدون حساسیت به حروف بزرگ و کوچک.

#### پاسخ
```bash
grep -i "network" system.log
```

#### توضیح
دستور `grep` برای جستجو در محتوای فایل‌ها به کار می‌رود. آپشن `-i` به معنی `case-insensitive` است، یعنی بین حروف بزرگ و کوچک فرق نمی‌کند. بنابراین این دستور تمام خطوطی را نشان می‌دهد که کلمه `network` را دارند، چه بزرگ باشد چه کوچک.

---

### تمرین 4: شمارش تعداد خطوط
با استفاده از `wc`، تعداد کل خطوط فایل `system.log` را مشخص کنید.

#### پاسخ
```bash
wc -l system.log
```

#### توضیح
دستور `wc` مخفف `Word Count` است و برای شمارش خطوط، کلمات و کاراکتر‌ها به کار می‌رود. آپشن `-l` فقط تعداد خطوط را نشان می‌دهد.

---

### تمرین 5: ترکیب grep و wc با pipe
چند خط در این فایل شامل کلمه `error` است؟

#### پاسخ
```bash
grep "error" system.log | wc -l
```

#### توضیح
این تمرین نشان می‌دهد که چگونه می‌توانیم از **piping** استفاده کنیم. در جلسه یاد گرفتیم که پایپ (`|`) خروجی استاندارد (stdout) یک دستور را به ورودی استاندارد (stdin) دستور بعدی می‌فرستد.

در این مثال:
1. `grep "error" system.log` تمام خطوطی را پیدا می‌کند که کلمه `error` دارند
2. `|` این خطوط را به دستور بعدی می‌فرستد
3. `wc -l` این خطوط را می‌شمارد و تعداد آن‌ها را نشان می‌دهد

اگر بخواهیم بدون حساسیت به حروف بزرگ و کوچک:
```bash
grep -i "error" system.log | wc -l
```

## تمرین‌های هفته آینده و پاسخ‌های تفصیلی

### سوال 1: آپشن شمارش خطوط در grep
با استفاده از `man grep`، آپشنی را پیدا کنید که به جای نمایش خطوط منطبق، فقط تعداد آنها را چاپ کند.

#### پاسخ
```bash
grep -c "pattern" file
```

#### توضیح
آپشن `-c` باعث می‌شود `grep` فقط تعداد خطوط منطبق را چاپ کند. مثال عملی:

```bash
grep -c "error" system.log
```

این دستور نتیجه‌ای مشابه `grep "error" system.log | wc -l` دارد، اما خیلی ساده‌تر است.

---

### سوال 2: پیدا کردن فایل‌های conf در /etc
دستوری بنویسید که تمام فایل‌های با پسوند `conf` را در کل دایرکتوری `/etc` و زیرشاخه‌های آن پیدا کند.

#### پاسخ
```bash
find /etc -type f -name "*.conf"
```

#### توضیح
دستور `find` برای جستجوی فایل‌ها و دایرکتوری‌ها استفاده می‌شود. در این دستور:

- `/etc`: دایرکتوری جستجو (و تمام زیرشاخه‌های آن)
- `-type f`: فقط فایل‌ها را جستجو کن (نه دایرکتوری‌ها)
- `-name "*.conf"`: فایل‌هایی که نام آن‌ها با `conf.` ختم می‌شود (استفاده از **wildcard** `*.conf`)

اگر جستجو بدون حساسیت به حروف بزرگ و کوچک باشد:
```bash
find /etc -type f -iname "*.conf"
```

### سوال 3: نمایش 100 خط از وسط فایل
فرض کنید یک فایل بزرگ به نام `data.csv` دارید. چگونه می‌توانید 100 خط از وسط این فایل (مثلاً از خط 1000 تا 1100) را مشاهده کنید؟

#### پاسخ (روش 1: با head و tail و pipe)
```bash
head -n 1100 data.csv | tail -n 100
```

#### توضیح
این روش از ترکیب `head` و `tail` استفاده می‌کند:

1. `head -n 1100` اولین 1100 خط را می‌گیرد
2. `|` این خطوط را به دستور بعدی می‌فرستد
3. `tail -n 100` آخرین 100 خط از آن 1100 خط را نمایش می‌دهد

نتیجه: خطوط 1001 تا 1100

#### پاسخ (روش 2: با sed - روش بهتر)
```bash
sed -n '1000,1100p' data.csv
```

#### توضیح
دستور `sed` (Stream Editor) راه‌حل بهتری است:
- `1000,1100`: از خط 1000 تا 1100
- `p`: چاپ خطوط مورد نظر
- `-n`: بدون چاپ خطوط دیگر

این دستور فقط خطوط درخواستی را نمایش می‌دهد و کارآمدتر است.

## نکات کلیدی جلسه

1. **Wildcards در شل**: برای نام‌گذاری فایل‌ها (مثل `*.log`)
2. **Regular Expressions در grep**: برای جستجو در محتوای فایل‌ها
3. **جریان‌های استاندارد**: stdin، stdout، stderr
4. **Piping**: ارسال خروجی یک دستور به ورودی دستور بعدی
5. **دستورات مهم**: `cat`، `less`، `head`، `tail`، `grep`، `find`، `wc`
