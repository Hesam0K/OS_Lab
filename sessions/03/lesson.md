# جلسه 3 - جستجو و کار با محتوای فایل‌ها

## بخش اول: مقدمه و مباحث تئوری

### 1. مرور سریع جلسه قبل
در جلسه قبل با مفاهیمی زیر آشنا شدیم:
- مدیریت فایل‌ها و دایرکتوری‌ها
- ساخت دایرکتوری با `mkdir`
- کپی، جابجایی و حذف فایل‌ها با `cp`، `mv` و `rm`
- ایجاد لینک سخت و لینک نرم با `ln`
- ویرایش فایل با `vim`

### 2. Wildcards چیست؟
Wildcards یا کاراکترهای جانشین، الگوهایی هستند که به شل کمک می‌کنند فایل‌ها را بر اساس نام پیدا کند.

#### کاراکترهای مهم
- `*`: جایگزین صفر یا چند کاراکتر
  - `*.txt`: همه فایل‌هایی که پسوند `txt` دارند
  - `file*`: همه فایل‌هایی که با `file` شروع می‌شوند
- `?`: جایگزین دقیقاً یک کاراکتر
  - `file?.txt`: فایل‌هایی مانند `file1.txt` یا `fileA.txt`
- `[]`: یکی از کاراکترهای داخل براکت را انتخاب می‌کند
  - `[abc]*.log`: فایل‌هایی که با `a` یا `b` یا `c` شروع شده‌اند
  - `file[1-5].c`: فایل‌های `file1.c` تا `file5.c`

### 3. تفاوت Wildcards با Regular Expressions
این دو مفهوم با هم اشتباه گرفته می‌شوند:
- Wildcards برای نام فایل‌ها و کار با شل استفاده می‌شوند.
- Regular Expressions برای جستجو در متن و یافتن الگوها در محتویات فایل‌ها استفاده می‌شوند.

مثال:
- `ls *.log` → استفاده از Wildcard برای پیدا کردن فایل‌ها
- `grep "error" system.log` → استفاده از Regex برای پیدا کردن متن در فایل

### 4. جریان‌های استاندارد در لینوکس
هر دستور در لینوکس سه جریان استاندارد دارد:
- `stdin`: ورودی استاندارد، معمولاً از کیبورد
- `stdout`: خروجی استاندارد، معمولاً در ترمینال نمایش داده می‌شود
- `stderr`: خروجی خطاها

### 5. Redirection و Piping
#### Redirection
با `>` می‌توان خروجی یک دستور را به جای ترمینال در یک فایل ذخیره کرد.

```bash
ls > file_list.txt
```

#### Piping
با `|` خروجی یک دستور را به ورودی دستور بعدی می‌دهیم.

```bash
grep "error" system.log | wc -l
```

### 6. کار با فایل‌های متنی بزرگ
برای فایل‌های بزرگ، استفاده از ویرایشگرهای گرافیکی ممکن است سنگین باشد. بنابراین از ابزارهای خط فرمان استفاده می‌کنیم.

## بخش دوم: کار عملی با دستورات

### 1. مشاهده محتوای فایل
#### `cat`
برای نمایش کامل محتویات فایل در ترمینال.

```bash
cat info.txt
cat file1.txt file2.txt > combined.txt
```

#### `less`
برای دیدن فایل صفحه‌به‌صفحه.

```bash
less system.log
```

کلیدهای مهم:
- `Space` یا `f`: صفحه بعد
- `b`: صفحه قبل
- `g`: رفتن به ابتدای فایل
- `G`: رفتن به انتهای فایل
- `/pattern`: جستجو
- `q`: خروج

#### `head` و `tail`
- `head`: نمایش چند خط اول
- `tail`: نمایش چند خط آخر

```bash
head system.log
head -n 20 system.log

tail system.log
tail -n 50 system.log
tail -f system.log
```

### 2. جستجوی فایل‌ها
#### `find`
برای پیدا کردن فایل‌ها و دایرکتوری‌ها بر اساس نام، نوع و ویژگی‌ها.

```bash
find . -name "*.log"
find . -iname "readme.md"
find /home -type d -name "Documents"
```

#### `which` و `whereis`
- `which`: مسیر اجرای یک دستور را نشان می‌دهد
- `whereis`: علاوه بر مسیر دستور، مسیر فایل راهنما را هم نشان می‌دهد

```bash
which ls
whereis ls
```

### 3. جستجو در محتوای فایل‌ها
#### `grep`
برای پیدا کردن خطوطی که حاوی یک الگوی مشخص هستند.

```bash
grep "error" system.log
grep -i "error" system.log
grep -v "error" system.log
grep -n "error" system.log
grep -r "error" /var/log/
```

### 4. شمارش
#### `wc`
برای شمارش خطوط، کلمات و کاراکترها.

```bash
wc system.log
wc -l system.log
wc -w system.log
wc -c system.log
```
