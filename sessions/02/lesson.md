# جلسه 2 - مدیریت فایل‌ها و دایرکتوری‌ها

## بخش اول: مقدمه و مباحث تئوری

### 1. مرور سریع جلسه قبل
در جلسه قبل با مفاهیمی مانند:
- `man`
- `cd`
- `ls`
- `pwd`

آشنا شدیم و با ساختار دایرکتوری‌های لینوکس و دستورات اولیه خط فرمان کار کردیم.

### 2. مفهوم فایل و دایرکتوری در لینوکس
در لینوکس، همه‌چیز یک فایل است؛ این جمله به این معناست که حتی دستگاه‌های سخت‌افزاری نیز در سطح سیستم‌عامل به‌صورت فایل‌های خاص نمایان می‌شوند.

- فایل: محلی برای ذخیره‌سازی داده‌ها مانند متن، تصویر، برنامه اجرایی و...
- دایرکتوری: فایلی خاص که شامل لیست فایل‌ها و دایرکتوری‌های دیگر است.

### 3. مفهوم لینک‌ها در لینوکس
#### inode چیست؟
`inode` یک شناسه‌ی منحصربه‌فرد برای هر فایل یا دایرکتوری در سیستم فایل است. این شناسه اطلاعاتی مانند مالک، سطح دسترسی و محل ذخیره‌سازی داده روی دیسک را نگه می‌دارد.

برای دیدن شماره inode می‌توان از دستور زیر استفاده کرد:

```bash
ls -i
```

#### لینک سخت (Hard Link)
- یک نام جدید برای همان inode ایجاد می‌کند.
- مانند دو اسم برای یک فایل است.
- اگر فایل اصلی حذف شود، داده‌ها تا زمانی که حداقل یک hard link باقی مانده باشد، پاک نمی‌شوند.
- فقط برای فایل‌ها و فقط در همان فایل سیستم قابل ایجاد است.

#### لینک نرم یا سمبولیک (Soft Link)
- یک فایل جدید ایجاد می‌کند که محتویات آن آدرس فایل اصلی است.
- مانند یک شورت‌کات در ویندوز عمل می‌کند.
- اگر فایل اصلی حذف شود، لینک نرم شکسته می‌شود.
- برای فایل‌ها و دایرکتوری‌ها و حتی بین فایل سیستم‌های مختلف قابل ایجاد است.

### 4. آشنایی با ویرایشگرهای متنی خط فرمان
برای ویرایش فایل‌های پیکربندی، نوشتن اسکریپت و برنامه‌نویسی بدون محیط گرافیکی، از ویرایشگرهای متنی استفاده می‌شود.

ویرایشگرهای رایج:
- `nano`: ساده و مناسب برای مبتدیان
- `emacs`: قدرتمند اما پیچیده
- `vim`: قدرتمند، سریع و روی اکثر سیستم‌های لینوکس موجود است

در این درس، با `vim` آشنا می‌شویم.

## بخش دوم: کار عملی با دستورات

### 1. ایجاد دایرکتوری
دستور `mkdir` برای ساخت دایرکتوری استفاده می‌شود.

```bash
mkdir my_project
mkdir docs images scripts
mkdir -p my_project/src/components
```

### 2. ایجاد فایل خالی
دستور `touch` برای ساخت فایل خالی یا به‌روزرسانی زمان دسترسی یک فایل استفاده می‌شود.

```bash
cd my_project
touch README.md main.c
```

### 3. کپی کردن فایل‌ها و دایرکتوری‌ها
دستور `cp` برای کپی کردن فایل‌ها و دایرکتوری‌ها استفاده می‌شود.

```bash
cp main.c main_backup.c
cp README.md ../docs/
cp -r ../scripts .
```

### 4. جابجایی و تغییر نام
دستور `mv` هم برای جابجایی و هم برای تغییر نام فایل‌ها و دایرکتوری‌ها به کار می‌رود.

```bash
mv main_backup.c old_main.c
mv main.c src/
```

### 5. حذف فایل‌ها و دایرکتوری‌ها
- `rmdir`: فقط دایرکتوری‌های خالی را حذف می‌کند.
- `rm`: برای حذف فایل‌ها و دایرکتوری‌ها.

```bash
rmdir images
rm old_main.c
rm -r scripts
rm -i README.md
```

### 6. ایجاد لینک‌ها
دستور `ln` برای ساخت hard link و symbolic link به کار می‌رود.

```bash
cd src
ln main.c main_hardlink.c
ln -s main.c main_softlink.c
ls -li
```

### 7. ویرایش فایل با vim
برای باز کردن فایل با vim:

```bash
vim main.c
```

#### حالت‌های اصلی vim
- `Normal Mode`: حالت پیش‌فرض برای اجرا دستورات
- `Insert Mode`: برای نوشتن متن، با `i` وارد می‌شود
- `Command Mode`: با `:` شروع می‌شود

#### دستورات مهم در vim
- `i`: ورود به حالت نوشتن
- `Esc`: خروج از حالت نوشتن
- `:w`: ذخیره فایل
- `:q`: خروج از vim
- `:wq` یا `:x`: ذخیره و خروج
- `:q!`: خروج بدون ذخیره
- `:w filename`: ذخیره با نام جدید

مثال ساده:

```c
#include <stdio.h>

int main(void) {
    printf("Hello, World!\n");
    return 0;
}
```

### 8. دستور `cat`
`cat` برای نمایش محتویات فایل روی ترمینال استفاده می‌شود.

```bash
cat main.c
cat file1.txt file2.txt
cat -n main.c
```
