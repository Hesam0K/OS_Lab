# جلسه 5 - مدیریت فرآیندها (Processes)

## بخش اول: مقدمه و مباحث تئوری

### 1. فرآیند (Process) چیست؟

**تعریف ساده:** فرآیند یک برنامه در حال اجرا است.

**آنالوژی معروف:** یک برنامه کامپیوتری مانند یک کتاب آشپزی است (مجموعه دستورالعملها). وقتی یک آشپز (CPU) شروع به خواندن کتاب و انجام مراحل آن می‌کند، یک فرآیند ایجاد شده است. می‌توان همزمان چندین آشپز از روی همان کتاب، کیک‌های متفاوتی بپزند (چندین فرآیند از یک برنامه).

### 2. مشخصات کلیدی یک فرآیند

- **PID (Process ID)**: شماره منحصربه‌فرد که سیستم‌عامل به هر فرآیند اختصاص می‌دهد
- **PPID (Parent Process ID)**: شناسه فرآیندی که این فرآیند را ایجاد کرده است
- **درخت فرآیندها**: تمام فرآیندهای سیستم (به جز اولین فرآیند با PID=1) توسط یک فرآیند دیگر ایجاد شده‌اند

### 3. فرآیندهای پیشزمینه و پسزمینه

**فرآیند پیشزمینه (Foreground):**
- کنترل ترمینال را در دست می‌گیرد
- شما مستقیماً با آن در تعامل هستید
- تا اجرای آن تمام نشود، نمی‌توانید دستور جدیدی وارد کنید

**فرآیند پسزمینه (Background):**
- بدون اشغال کردن ترمینال اجرا می‌شود
- می‌توانید همزمان کار خود را ادامه دهید
- مناسب برای کارهای زمانبر

### 4. جاب (Job) و سیگنال (Signal)

**جاب:** یک "کار" یا فرآیندی است که از طریق شل اجرا شده است. شل به هر جاب شماره‌ای اختصاص می‌دهد.

**سیگنال:** پیغامی کوتاه است که سیستم‌عامل یا یک فرآیند برای یک فرآیند دیگر ارسال می‌کند.

#### سیگنال‌های معروف
- **SIGINT**: با `Ctrl+C` ارسال می‌شود. از فرآیند درخواست خاتمه
- **SIGTSTP**: با `Ctrl+Z` ارسال می‌شود. فرآیند را متوقف (suspend) می‌کند
- **SIGTERM**: سیگنال پیش‌فرض `kill`. درخواست محترمانه برای خاتمه
- **SIGKILL**: سیگنال شماره 9. دستور قتل فوری بدون پاکسازی

## بخش دوم: کار عملی با دستورات

### 1. مشاهده فرآیندها

#### `ps` - Process Status
```bash
ps              # فرآیندهای ترمینال فعلی
ps aux          # تمام فرآیندها با جزئیات کامل
ps -ef          # فرمت دیگری برای نمایش تمام فرآیندها
```

#### `pstree` - درخت فرآیندها
```bash
pstree          # نمایش درختی فرآیندها
```

#### `top` - داشبورد زنده
```bash
top             # مانند Task Manager ویندوز
# برای خروج: q
```

### 2. مدیریت جاب‌ها

#### اجرای فرآیند در پسزمینه
```bash
sleep 200 &     # اضافه کردن & به انتهای دستور
```

#### `jobs` - لیست جاب‌های فعال
```bash
jobs            # نمایش جاب‌های در حال اجرا یا متوقف‌شده
```

#### متوقف کردن فرآیند
```bash
Ctrl+Z          # متوقف‌کردن فرآیند پیشزمینه
```

#### `fg` - آوردن به پیشزمینه
```bash
fg %1           # جاب شماره 1 را به پیشزمینه بیاور
```

#### `bg` - ادامه در پسزمینه
```bash
bg %1           # جاب متوقف‌شده شماره 1 را در پسزمینه ادامه بده
```

### 3. ارسال سیگنال و خاتمه دادن

#### `kill` - ارسال سیگنال
```bash
kill %1         # خاتمه دادن به جاب شماره 1 (SIGTERM)
kill 12345      # خاتمه دادن با استفاده از PID
kill -9 12345   # SIGKILL - خاتمه فوری
kill -SIGKILL 12345
```

#### `killall` - خاتمه تمام فرآیندهای با یک نام
```bash
killall firefox # تمام فرآیندهای firefox را خاتمه بده
```

## بخش سوم: فلسفه یونیکس و جریان‌های استاندارد

### فلسفه یونیکس

- **قانون اصلی**: برنامه‌ها باید یک کار را انجام دهند و آن را به خوبی انجام دهند
- **قانون ترکیب**: برنامه‌ها باید طوری نوشته شوند که بتوانند با یکدیگر کار کنند
- **آنالوژی**: ابزارهای لینوکس مانند لگو (Lego) هستند

### جریان‌های استاندارد

- **stdin** (شماره 0): ورودی پیش‌فرض (معمولاً کیبورد)
- **stdout** (شماره 1): خروجی موفقیت‌آمیز
- **stderr** (شماره 2): خروجی خطاها

### تغییر مسیر (Redirection)

```bash
ls > file.txt           # stdout را به فایل هدایت کن
ls >> file.txt          # stdout را به انتهای فایل اضافه کن
ls 2> errors.txt        # stderr را به فایل هدایت کن
ls &> output.txt        # هر دو را به فایل هدایت کن
ls > /dev/null          # خروجی را نادیده بگیر
```

### لولهکشی (Piping)

```bash
ps aux | grep firefox   # خروجی ps را به grep بده
ls /etc/ | wc -l        # تعداد فایل‌ها را بشمار
history | grep "git" | tail -n 10
```
