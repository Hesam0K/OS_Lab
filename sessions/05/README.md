# جلسه 5 - مدیریت فرآیندها (Processes)

## تمرین‌های عملی در کلاس و پاسخ‌های کامل

### تمرین 1: اجرای دستور در پسزمینه
یک دستور `sleep 500` را در پسزمینه اجرا کنید.

#### پاسخ
```bash
sleep 500 &
```

**نتیجه انتظاری:**
```
[1] 12345
```

این به معنی این است که:
- جاب شماره 1 است
- PID (شناسه فرآیند) 12345 است

---

### تمرین 2: یافتن PID فرآیند
با استفاده از `ps aux | grep sleep` شناسه فرآیند (PID) آن را پیدا کنید.

#### پاسخ
```bash
ps aux | grep sleep
```

**توضیح:**
- `ps aux` تمام فرآیندها را با جزئیات کامل نمایش می‌دهد
- `|` (pipe) خروجی را به دستور بعدی می‌فرستد
- `grep sleep` فقط خطوطی را نشان می‌دهد که کلمه `sleep` دارند

**نتیجه انتظاری:**
```
user  12345  0.0  0.0  5180  380 pts/0  S  10:30  0:00  sleep 500
```

---

### تمرین 3: متوقف‌کردن فرآیند
یک دستور `ping 8.8.8.8` را در پیشزمینه اجرا کرده و سپس با `Ctrl+Z` آن را متوقف کنید.

#### پاسخ
```bash
ping 8.8.8.8
# سپس Ctrl+Z را بزنید
```

**توضیح:**
- در جلسه یاد گرفتیم که `Ctrl+Z` سیگنال `SIGTSTP` ارسال می‌کند
- این سیگنال فرآیند را متوقف (suspend) می‌کند، اما آن را از بین نمی‌برد

**نتیجه انتظاری:**
```
^Z
[2]+  Stopped                 ping 8.8.8.8
```

---

### تمرین 4: مشاهده لیست جاب‌ها
با دستور `jobs` لیست جاب‌های فعال را ببینید.

#### پاسخ
```bash
jobs
```

**توضیح:**
دستور `jobs` تمام جاب‌های در حال اجرا یا متوقف‌شده را نمایش می‌دهد.

**نتیجه انتظاری:**
```
[1]-  Running                 sleep 500 &
[2]+  Stopped                 ping 8.8.8.8
```

---

### تمرین 5: ادامه فرآیند در پسزمینه
جاب `ping` را در پسزمینه ادامه دهید (bg).

#### پاسخ
```bash
bg %2
```

**توضیح:**
- دستور `bg` جاب متوقف‌شده را در پسزمینه ادامه می‌دهد
- `%2` به معنی جاب شماره 2 است

**نتیجه انتظاری:**
```
[2]+ ping 8.8.8.8 &
```

---

### تمرین 6: آوردن فرآیند به پیشزمینه و خاتمه
جاب `sleep` را به پیشزمینه بیاورید (fg) و سپس با `Ctrl+C` آن را خاتمه دهید.

#### پاسخ
```bash
fg %1
# سپس Ctrl+C را بزنید
```

**توضیح:**
- `fg` جاب را از پسزمینه به پیشزمینه می‌آورد
- `Ctrl+C` سیگنال `SIGINT` ارسال می‌کند و فرآیند را خاتمه می‌دهد

---

### تمرین 7: خاتمه فرآیند با kill
جاب `ping` را با دستور `kill` و شماره جاب آن خاتمه دهید.

#### پاسخ
```bash
kill %2
```

**توضیح:**
- `kill` به طور پیش‌فرض سیگنال `SIGTERM` ارسال می‌کند
- این درخواست محترمانه برای خاتمه فرآیند است

یا می‌توانید با PID استفاده کنید:
```bash
kill 12346
```

### تمرین 8: ذخیره فایل‌های مخفی
دستوری بنویسید که لیست تمام فایل‌های مخفی در دایرکتوری خانه شما را در فایلی به نام `hidden_files.txt` ذخیره کند.

#### پاسخ
```bash
ls -a ~/ | grep "^\." > hidden_files.txt
```

**توضیح:**
- `ls -a ~/`: تمام فایل‌ها (شامل مخفی‌ها) در دایرکتوری خانه
- `grep "^\."`: فقط فایل‌هایی که با نقطه شروع می‌شوند (مخفی)
- `>`: خروجی را به فایل هدایت می‌کند

یا روش ساده‌تر:
```bash
ls -d ~/.* > hidden_files.txt 2>/dev/null
```

### تمرین 9: یافتن فرآیندهای پرمصرف
دستوری بنویسید که 5 فرآیندی که بیشترین مصرف حافظه دارند، پیدا کند.

#### پاسخ
```bash
ps aux | sort -nk4 -r | head -n 5
```

**توضیح:**
- `ps aux`: نمایش تمام فرآیندها
- `sort -nk4 -r`: مرتب‌سازی عددی (`-n`) بر اساس ستون 4 (`-k4`) به‌صورت نزولی (`-r`)
  - ستون 4 درصد مصرف حافظه است
- `head -n 5`: نمایش 5 خط اول

### تمرین 10: مرتب‌سازی و حذف موارد تکراری
یک فایل `names.txt` با محتوای زیر بسازید:

```
ali
sara
reza
ali
maryam
sara
```

سپس دستوری بنویسید که اسامی را مرتب کرده، موارد تکراری را حذف کند و نتیجه را نمایش دهد.

#### پاسخ (ایجاد فایل)
```bash
cat > names.txt << EOF
ali
sara
reza
ali
maryam
sara
EOF
```

یا با vim:
```bash
vim names.txt
# سپس اسامی را وارد کنید و `:wq` بزنید
```

#### پاسخ (مرتب‌سازی و حذف تکراری)
```bash
sort names.txt | uniq
```

**توضیح:**
- `sort names.txt`: فایل را مرتب می‌کند
- `|`: خروجی را به دستور بعدی می‌فرستد
- `uniq`: موارد تکراری متوالی را حذف می‌کند

**نتیجه انتظاری:**
```
ali
maryam
reza
sara
```

## تمرین‌های هفته آینده و پاسخ‌های تفصیلی

### سوال 1: PID اول و systemd
PID اولین فرآیندی که در سیستم شما اجرا می‌شود (معروف به init یا systemd) چیست و چرا این فرآیند والد تمام فرآیندهای دیگر محسوب می‌شود؟

#### پاسخ
```bash
ps aux | head -n 2
# یا برای دیدن درخت کامل:
pstree
```

**نتیجه انتظاری:**
```
UID  PID  PPID ... COMMAND
root   1      0 ... /lib/systemd/systemd
```

**توضیح:**
- **PID اول**: معمولاً 1 است
- **فرآیندی**: `systemd` یا `init` در سیستم‌های قدیمی
- **چرا والد تمام فرآیندها**: در جلسه یاد گرفتیم که هر فرآیند (به جز فرآیند اول) توسط یک فرآیند دیگر ایجاد می‌شود. `systemd` اولین فرآیند است و بقیه در سلسله‌مراتب زیر آن ایجاد می‌شوند.
- **PPID صفر**: فرآیند شماره 1 PPID برابر 0 دارد چون هیچ والد ندارد.

---

### سوال 2: شماره سیگنال SIGTSTP
با استفاده از `man kill` یا `kill -l`، شماره سیگنال SIGTSTP چیست؟

#### پاسخ
```bash
kill -l | grep SIGTSTP
```

**نتیجه انتظاری:**
```
20) SIGTSTP
```

**توضیح:**
- شماره `SIGTSTP` برابر 20 است
- این سیگنال معادل `Ctrl+Z` است
- در جلسه یاد گرفتیم که این سیگنال فرآیند را متوقف (suspend) می‌کند

### سوال 3: تفاوت SIGTERM و SIGKILL
تفاوت اصلی بین `SIGTERM` و `SIGKILL` چیست؟ در چه شرایطی از `SIGKILL` استفاده می‌کنیم؟

#### پاسخ

| ویژگی | SIGTERM | SIGKILL |
|------|---------|---------|
| شماره | 15 | 9 |
| نوع | درخواست محترمانه | دستور فوری |
| قابل بلاک | بله | خیر |
| اجازه پاکسازی | بله | خیر |
| استفاده پیش‌فرض | kill 12345 | kill -9 12345 |

**توضیح:**
- **SIGTERM** (سیگنال 15):
  - درخواست محترمانه برای خاتمه فرآیند
  - فرآیند می‌تواند این سیگنال را نادیده بگیرد
  - فرآیند فرصت دارد کارهای پاکسازی انجام دهد

- **SIGKILL** (سیگنال 9):
  - دستور فوری و غیرقابل نادیده گذاری
  - فرآیند بالفاصله از بین می‌رود
  - بدون فرصت پاکسازی

**چه زمان از SIGKILL استفاده می‌کنیم:**
- وقتی فرآیند به `SIGTERM` پاسخ نمی‌دهد
- وقتی فرآیند در حلقه بی‌نهایت گیر افتاده است
- درمورد فرآیندهای مخرب

---

### سوال 4: Useless Use of Cat
تفاوت اصلی بین این دو دستور چیست؟

```bash
cat file.txt | grep "word"
grep "word" file.txt
```

#### پاسخ

**دستور اول:**
```bash
cat file.txt | grep "word"
```
- 2 فرآیند: `cat` و `grep`
- `cat` محتوای فایل را خوانده و به `grep` می‌فرستد

**دستور دوم:**
```bash
grep "word" file.txt
```
- 1 فرآیند: فقط `grep`
- `grep` مستقیماً فایل را باز می‌کند

**نتیجه یکسان است، اما دستور دوم بهتر است.**

**توضیح:**
- این حالت "Useless Use of Cat" نام دارد
- دستور اول فرآیند اضافی ایجاد می‌کند
- دستور دوم کارآمد‌تر است چون `grep` می‌تواند مستقیماً فایل را بخواند

### سوال 5: فیلتر کردن خطاهای کرنل
دستوری بنویسید که تمام پیغامهای مربوط به کرنل سیستم را گرفته (dmesg)، فقط خطوطی که شامل کلمه `error` یا `fail` هستند را فیلتر کرده و نتیجه را در فایل `kernel_errors.log` ذخیره کند.

#### پاسخ
```bash
dmesg | grep -i "error\|fail" > kernel_errors.log
```

**توضیح:**
- `dmesg`: پیغامهای کرنل را نمایش می‌دهد
- `grep -i "error\|fail"`: 
  - `-i`: case-insensitive (بدون حساسیت به حروف بزرگ/کوچک)
  - `"error\|fail"`: خطوطی که `error` یا `fail` دارند
- `>`: خروجی را به فایل هدایت می‌کند

یا برای جدا کردن stderr:
```bash
dmesg 2>&1 | grep -i "error\|fail" > kernel_errors.log
```

### سوال 6: دستور tr
دستور `tr` چیست و چه کاری انجام می‌دهد؟ یک مثال از ترکیب `tr`، `cat` و `sort` با pipe ارائه دهید.

#### پاسخ

**دستور tr چیست:**
```bash
man tr
```

**توضیح:**
- `tr` مخفف `translate` است
- حروف یا رشته‌های متن را تغییر می‌دهد
- استفاده: `tr [options] SET1 [SET2]`

**مثال 1: تبدیل حروف کوچک به بزرگ**
```bash
cat names.txt | tr 'a-z' 'A-Z'
```

**مثال 2: حذف اعداد**
```bash
cat file.txt | tr -d '0-9'
```

**مثال 3: ترکیب tr، cat و sort**
```bash
cat names.txt | tr '[:upper:]' '[:lower:]' | sort
```

**توضیح:**
- `cat names.txt`: محتوای فایل را خوانده
- `tr '[:upper:]' '[:lower:]'`: تمام حروف بزرگ را به کوچک تبدیل می‌کند
- `sort`: نتیجه را مرتب می‌کند

**نتیجه انتظاری (اگر فایل ali, SARA, reza داشته باشد):**
```
ali
reza
sara
```

## خلاصه دستورات مهم

| دستور | توضیح |
|-------|-------|
| `ps aux` | نمایش تمام فرآیندها |
| `top` | داشبورد زنده سیستم |
| `pstree` | نمایش درختی فرآیندها |
| `jobs` | لیست جاب‌های فعال |
| `fg %1` | آوردن جاب به پیشزمینه |
| `bg %1` | ادامه جاب در پسزمینه |
| `kill %1` | خاتمه جاب |
| `kill -9 PID` | خاتمه فوری فرآیند |

1. **فلسفه یونیکس**: هر برنامه یک کار را خوبی انجام دهد

3. **سیگنال‌ها**: ابزار مهم برای ارتباط با فرآیندها
